# 申请证明材料准备指南

为了提高 Token Plan 申请通过率，建议准备以下材料。

## 推荐材料

### 1. GitHub 仓库

仓库需要包含：

- 清晰的 README。
- 项目背景和痛点说明。
- 可运行的代码或原型。
- 配置示例。
- 后续计划。

### 2. 终端截图

建议截图内容：

- Claude Code / OpenCode / OpenClaw 启动界面。
- Agent 执行真实任务的过程。
- Agent 分析错误日志或生成配置的过程。

避免只上传失败截图。失败截图可以作为排错过程的一部分，但最好搭配成功运行截图。

### 3. 配置截图

可以上传 CC Switch 或类似工具的配置页面，证明项目确实围绕多模型供应商接入展开。

截图前请隐藏：

- API Key
- 个人手机号
- 个人邮箱
- 账单流水
- 真实访问令牌

### 4. 录屏

录屏建议控制在 30 秒到 2 分钟，展示：

1. 打开项目仓库；
2. 运行 CLI 工具；
3. 生成配置或解释错误；
4. 在 Claude Code 中继续使用 Agent 排查或开发。

## 适合填写到申请表的项目标题

基于 Claude Code 与 CC Switch 的多模型 Agent Coding 工作流

## 适合填写到申请表的项目链接说明

这是一个面向个人开发者和小团队的 AI Coding / Agent 多模型供应商接入工具原型，用于解决 Claude Code、OpenClaw、OpenCode 等工具接入不同模型供应商时的配置复杂、协议差异和错误排查问题。项目包含供应商配置模板、环境变量生成、常见错误码诊断和后续多 Agent 工作流设计。
